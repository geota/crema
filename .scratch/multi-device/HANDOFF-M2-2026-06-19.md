**Architecture in one breath:** managers sit on a `SwitchableBleTransport` facade. NORMAL = `NordicBleTransport`. PRIMARY = `TappingBleTransport`(real or `ReplayBleTransport`) → `RelayHub` + `LanRelayServer` (Ktor WS). SECONDARY = `ProxyTransport` over `ReconnectingClientLink`. The picker drives a **live** `applyMode` swap (disconnect→setDelegate→reconnect, no restart). A secondary's core is **read-only** (zero writes); its user taps **relay** as `Control` frames to the primary's VM router; the primary pushes a **`Config` snapshot** on attach so the mirror's config can't drift.

## Remaining work (this session's targets: T2-polish + M3)

### T2 polish (small — same patterns as T1b/T2)
1. **Live `pushConfig` hooks.** `RelayHub.pushConfig()` exists but isn't called on the primary's mid-session config changes — only the attach-time snapshot fires. Hook `relayHub?.pushConfig()` where the primary's config changes (cleanest single point: in `persistPrefs()` if all config setters route through it — verify; else the unit/profile/QC setters). `relayHub` is null off-primary so the call is a safe no-op there.
2. **Relay the config-authority verbs.** `setActiveProfile`/`setActiveBean`/`setStopOnWeight`/`quickAdjustBrew`/`setQc*` are NOT relayed — a secondary editing config diverges until re-attach. Wrap each with `relayIfSecondary("<verb>", args)` (the T1b helper) + add to `handleRelayedControl`'s `when`. Args: profile/bean id = the string; bool = `.toString()`; quickAdjustBrew = encode the 4 doubles (small @Serializable or CSV). The primary's setter then `pushConfig()`s the result back (composes with #1).
3. **Authority indicator.** A "Viewing as secondary / Mirroring <primary>" badge on the phone Brew screen. Today the only cues are the orange bluetooth icon + the Devices-sheet "Mirroring" row. `ConfigSnapshot.authority` already crosses; surface it.

### M3 — handoff (role-swap; the big one)
Move the radio + transfer session context, **idle-only** (mid-shot stays Mirror). New wire frames (see M1-PROTOCOL §11 open items + RESEARCH "Handoff"). Sketch: a `Handoff` request frame secondary→primary ("you take the DE1" / "give me the DE1"); the holder `disconnect()`s its real link, the taker `connect()`s; both `applyMode`-swap roles (primary↔secondary) live; the config travels with the role. **Gate to idle** (`machineState != Espresso/Steam/HotWater/Flush`). This needs its own careful design pass — it is NOT a quick add.

## Validation env + demo recipe (2 emulators)

- `adb` = `~/Library/Android/sdk/platform-tools/adb`; emulators started via MCP `start_emulator` (survive across turns; **do NOT** boot the emulator binary via Bash `run_in_background` — the sandbox kills GUI children). AVDs: `Medium_Tablet` (→ `emulator-5554`, primary, **renders at 2560×1600**; the MCP screenshot is downscaled to 1280×800, so tap by UI-tree coords), `Pixel_10_Pro` (→ `emulator-5556`, phone secondary, **1280×2856 native = tap coords match the screenshot**). Package `coffee.crema`, launcher `coffee.crema/.ui.MainActivity`.
- **Replay-primary recipe (no Bluetooth):** the good capture is staged at `/data/local/tmp/shot.jsonl` on the tablet. Re-trigger a fresh shot:
  1. `adb -s emulator-5554 shell am force-stop coffee.crema`
  2. clean + restage: `rm -f <ext>/captures/*.jsonl; cp /data/local/tmp/shot.jsonl <ext>/captures/session-…-shot-pull.jsonl; chmod 666 …` where `<ext>` = `/storage/emulated/0/Android/data/coffee.crema/files`. (Direct `adb push` into app-data fails `fchown`; stage via `/data/local/tmp` + `cp`.)
  3. relaunch; read the **ephemeral** relay port from logcat tag `LanRelayServer` ("LAN relay listening on :N"); `adb -s emulator-5554 forward tcp:8089 tcp:N`.
  4. Espresso is at replay **t+59s** (state 04); the phone dials `10.0.2.2:8089` → host:8089 → tablet:N.
- **Phone prefs (secondary) via run-as:** `echo '{"proxyRole":"secondary","proxyPrimaryHost":"10.0.2.2","proxyPrimaryPort":8089,"de1Address":"DE1:REPLAY","showDebugPanel":true}' | adb -s emulator-5556 shell "run-as coffee.crema sh -c 'cat > /data/data/coffee.crema/files/prefs.json'"`. **Tablet prefs already have psi/°F/oz** — that's why the T2 units-snap is visible (phone defaults bar/°C/g).
- **Gotchas:** `appendLog` goes to the **in-app event log, NOT logcat** (only `Log.i` reaches logcat — `LanRelayServer`, `RelayHub: relayed control:`, `ReconnectingClientLink: Connected to primary`, `ProxyTransport: write rejected`). Foreground `sleep >~10s` is blocked → use `wait_for_element` or an `until`-loop. The tablet "Stop" text matches "STOP CONDITIONS" — wait on "Pouring" for the espresso window.
