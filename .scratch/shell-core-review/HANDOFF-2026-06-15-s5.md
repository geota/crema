## Non-obvious things found this session (read before touching these)

- **`clearTokens()` DOES fire `onConnectionChange` (issue 39).** A stale comment in
  `SharingSection.disconnect` claimed "we own this transition; no listener". Not
  true: `TokenVault.clearTokens` does `Ref.set(changes, null)` on the same
  `SubscriptionRef` that `onConnectionChange` consumes, so a local sign-out *does*
  propagate — just a tick later (forked runtime fiber). The new helper's `connected`
  is therefore a **getter *and* setter**: SharingSection sets `viz.connected = false`
  for an *optimistic* snappy disconnect, and the subscription confirms it (no-op).
  The two delete-splits only read.
- **The connection gate had 4 copies, not 3 (issue 39).** The issue/punchlist said
  "3×"; the acceptance grep (`onConnectionChange` over `components/`) surfaced a 4th:
  `settings/sections/BeanSyncSection.svelte`. Always trust the grep over the issue's
  file list.
- **Rune helpers can own `bind:`-target state (issue 41).** `useStepper` exposes
  `draft` and `inputEl` as getter/**setter** lvalues, and `bind:value={stepper.draft}`
  / `bind:this={stepper.inputEl}` work — confirmed against the Svelte `bind:` docs
  (object-property bindings are valid lvalues) and live in-browser. This is the
  pattern for extracting a stepper/input state machine into `.svelte.ts`.
- **Pass props as getters or reactivity breaks (issue 41).** `useStepper` takes
  `value: () => value`, `step: () => step`, etc. `svelte-check` caught the one place
  I slipped (`commit: onCommit` captured the *initial* prop) — fixed to
  `(n) => onCommit(n)`. If you extract reactive logic into `.svelte.ts`, every prop
  read must be a thunk; trust `svelte-check`'s `state_referenced_locally` warning.
- **Issue 45 radius was a judgement call.** All three sub-items were Android drifting
  from web `tokens.css`, so all align **Android → web**. For the sheet radius I picked
  web's **24** (Android `extraLarge` 28dp → 24dp); 28 was the M3 standard, so if the
  softer sheet is wanted, flip web `--radius-xl` to 28px instead — one line. Also
  **found-but-deferred:** `surfaceContainerHigh`/paper-200 *also* drifted
  (`Color.kt` `0xFFECE2D1` vs web `#ECDFC8`) — outside #45's three sub-items.
