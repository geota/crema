## Non-obvious things found this session (read before touching these)

- **`beanFilterCounts` fixed a real bug (issue 28·2).** The phone "All" bean chip
  counted `ui.beans.size` (incl. archived) while the *list* hid archived — proven live
  (archive a bean → All 7→6, Archived 0→1, Restore → 7). Tablet was already correct.
- **History: the phone keeps its own `startOfDay`/`dayMs` (issue 28·3).** They also
  drive the `dayLabel` day-grouping (THIS WEEK / EARLIER), not just the filter — don't
  remove them when hoisting `filterAndSortShots`. Tablet's *did* become removable, and
  its pre-sort `filtered` (export count + `StatsStrip`, issue 48) was repointed to
  `shots` (same set, order-irrelevant).
- **`effectiveBrew` is the 3-way override→profile→default ONLY (issue 28·1).** The
  segment-temp default (`seg.temp ?: 93f`) and the 2-way profile-only recipe
  (`active?.dose ?: 18f`, tablet Brew/Scale) are *different concepts* — NOT folded. The
  acceptance grep `?: 18/36/93` is deliberately non-zero. PhoneScale "Add 0.5 g" keeps
  its own `bumpedDose * 2` yield. (Yield seed harmonised to `dose*2` per review.)
- **`CremaStepper` carries `compareSymbol`/`onCompare` for ALL compact styles
  (issue 35 → 50).** That's why issue 50-part1 was a 4-line change: the phone Exit
  threshold (`BareCompact`) just takes the same params the tablet (`BoxedDense`) uses.
- **`CremaStatusDot` + the steppers SHRANK issue 26.** 26's acceptance grep listed
  `SetStepper`/`PStepper`/`StatusDot`/`PStatusDot` — all already gone. 26 is now ONLY
  the row chrome + `PPill`/`PSelect`/`PMono` ↔ `SetPill`/`SetSelect`/`MonoReadout` +
  the dead `SettingsRow` (`CremaPhoneComponents.kt:409`). See 26's note.
- **Issue 32 exit-unit is `SegmentExit.unit()`, not `SegmentEdit.exitUnit()`.** The
  exit owns its metric and sites use a *defaulted* exit view (`exView = seg.exit ?: …`).
