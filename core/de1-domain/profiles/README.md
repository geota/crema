# Built-in DE1 profiles

`builtin.json` holds Crema's built-in profiles in Crema's **native** format: a
JSON array of `Profile` (the `de1-domain` model). It is embedded into the
`de1-domain` crate at compile time (`include_str!`) and exposed as built-in
Crema `Profile`s by the [`builtin`](../src/builtin.rs) module.

The profiles are the 88 standard espresso/tea/cleaning profiles shipped by the
**de1app** project — the original Decent Espresso DE1 tablet application
(`de1plus/profiles/`). Each upstream `*.tcl` file was converted once via
`de1_domain::import_legacy_tcl`, then all the resulting `Profile`s were
serialized into `builtin.json`. The upstream `.tcl` files are *not* vendored as
shipped data; `builtin.json` is the only embedded data file. (Two upstream `.tcl`
profiles are retained under `tests/fixtures/` purely as legacy-import parser test
inputs — see [`THIRD-PARTY-NOTICES.md`](../../../THIRD-PARTY-NOTICES.md).)

## Origin

- Upstream project: <https://github.com/decentespresso/de1app>
- Path in upstream: `de1plus/profiles/` (the `*.tcl` profile files)

## Regenerating

`builtin.json` is regenerated by importing every de1app `*.tcl` profile and
serializing the `Vec<Profile>` to JSON: run `import_legacy_tcl` over each file
in the upstream `de1plus/profiles/` directory (sorted by file name) and write
`serde_json::to_string_pretty(&profiles)` to this file.
